# pouch

`pouch` は、パスからファイルまたはディレクトリを作成する小さな CLI である。
足りないパスは作成するが、既存ファイルは変更しない。

auto モードでは、次の 1 つのルールで判定する。

- 最後のパスセグメントにドット (`.`) が含まれていれば、ファイルとして扱う。
- そうでなければ、ディレクトリとして扱う。

このルールにより、`mkdir -p` と `touch` を都度使い分けなくても、よくあるパスを作成できる。

## 例

```sh
pouch notes
pouch notes/today.md
pouch src/main.go test
```

それぞれの動作:

- `pouch notes` は `notes` ディレクトリを作成する。
- `pouch notes/today.md` は親ディレクトリを作成してから `today.md` を作成する。
- `pouch src/main.go test` は各パスを入力順に処理する。

## 目的

パスを作るだけでも、通常は複数のコマンドを使い分けることになる。

```sh
mkdir -p notes
mkdir -p src && touch src/main.go
```

`pouch` は、それを 1 つのコマンドと 1 つの判定ルールにまとめる。

## auto モードの判定

auto モードでは、最後のパスセグメントだけを見る。

例:

- `sample` -> ディレクトリ。
- `sample.ts` -> ファイル。
- `sample/temp.ts` -> ファイル。
- `.env` -> ファイル。

`pouch` は、このルールを意図的に小さく保つ。既知のファイル名、MIME type、末尾スラッシュから意図を推測しない。

## `--mode` を使う場面

auto ルールでは判定が分かれる名前がある。

- `Dockerfile` は auto モードではディレクトリになる。
- `Makefile` は auto モードではディレクトリになる。
- `dir.with.dot` は auto モードではファイルになる。

別の結果にしたい場合は `--mode` を使う。

```sh
pouch --mode file Dockerfile
pouch --mode dir dir.with.dot
```

## 振る舞い

### ファイルパス

`pouch` がパスをファイルとして扱う場合の動作:

1. 足りない親ディレクトリを作成する。
2. ファイルが存在しなければ作成する。
3. すでに存在する場合は、何も変更しない。
4. 対象パスがディレクトリとして存在する場合は、エラーにする。

### ディレクトリパス

`pouch` がパスをディレクトリとして扱う場合の動作:

1. `mkdir -p` 相当でディレクトリを作成する。
2. すでに存在する場合も成功として扱う。
3. 対象パスがファイルとして存在する場合は、エラーにする。

## CLI

基本形:

```sh
pouch [flags] PATH...
```

フラグ:

- `-m, --mode <auto|file|dir>`: ファイルかディレクトリかを明示する。
- `-n, --dry-run`: ファイルシステムを変更せず、予定している操作だけを表示する。
- `-v, --verbose`: 各操作を入力順に表示する。
- `-h, --help`: ヘルプを表示する。
- `--version`: バージョンを表示する。

## 終了挙動

- すべて成功したら終了コード `0` を返す。
- 最初のエラーで非ゼロ終了する。
- エラーは標準エラー出力へ書く。
- 複数パスを受け取った場合も入力順に処理し、最初のエラーで停止する。

## スコープ

`pouch` は意図的に対象を絞っている。

対応 OS:

- macOS
- Linux

含めるもの:

- CLI 引数からのパス作成。
- 単純な自動判定。
- 明示モードによる上書き。
- CLI とテストを支える Go パッケージ。

含めないもの:

- テンプレート生成。
- ファイル内容の生成。
- プロジェクトスキャフォールディング。
- 初回リリースでの設定ファイル。
- 初回リリースでの対話プロンプト。
- Windows 対応。
